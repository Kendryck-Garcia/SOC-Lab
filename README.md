SOC-Lab
1. Project Overview
This project involved building a controlled virtualized environment to simulate real-world cyber attacks and analyze the resulting security telemetry. By acting as both the Adversary (Kali Linux) and the Defender (Wazuh SIEM), I gained hands-on experience in identifying attack patterns, configuring endpoint detection, and investigating security incidents.
2. Tools & Technologies Used
•	SIEM: Wazuh
•	Endpoint Detection: Wazuh Agent & Windows Sysmon
•	Attack Platform: Kali Linux
•	Target Environment: Windows 11
•	Virtualization: VMware Workstation
•	Optional: Tailscale, Windows Sysmon
3. Deployment Steps
Step 1: Architecting the SOC Environment
I deployed a distributed lab architecture using a private NAT network to ensure a safe, isolated testing environment.
•	Installed the Wazuh OVA to act as the central security "brain."
•	Configured Windows 11 as the primary target (Victim).
•	Verified network connectivity between all nodes using ping and ip a.

Step 4: Incident Analysis & Detection (Defensive Operations)
I monitored the Wazuh Dashboard to analyze the alerts generated by the simulated attacks.
•	Investigated Logon Failure (Event 60122) alerts to identify the attacker's source IP.
•	Documented the timeline of the "Test" account creation.
•	Analyzed the severity levels of the alerts to prioritize the response.

4. Key Findings & Conclusion
Through this lab, I successfully demonstrated how an attacker's "noisy" actions (like brute-forcing) leave a clear trail in the system logs. This project reinforced my understanding of Log Management, Network Security, and the importance of Endpoint Visibility in a modern Security Operations Center.
